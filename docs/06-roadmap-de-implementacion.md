# Roadmap de implementacion

## Etapa 1 - Nucleo comun

Objetivo: tener una base estable para todos los tramites.

Entregables:

- usuarios, roles y permisos
- talleres y sucursales
- alta de casos
- identificador de carpeta
- ficha tecnica base
- adjuntos y categorias
- panel general inicial
- auditoria de creacion y cambios

## Etapa 2 - Presupuesto comun

Entregables:

- formulario de presupuesto reutilizable
- items de piezas, tareas y nivel de dano
- fotos y videos del vehiculo
- calculos automaticos
- cierre de informe
- exporte PDF

## Etapa 3 - Particular

Entregables:

- flujo completo de reparacion
- pagos y saldo deudor
- recibo PDF
- cierre automatico del caso

## Etapa 4 - Todo Riesgo y Granizo

Entregables:

- gestion del tramite
- franquicia
- prescripcion
- tramitacion ante compania
- reglas de acceso a reparacion

## Etapa 5 - Repuestos, turnos y agenda

Entregables:

- gestion de repuestos
- inventario por repuesto
- turnos con calculo de salida
- ingresos, egresos y reingresos
- agenda de tareas por usuario
- alertas al administrador

## Etapa 6 - Tramites complejos

Entregables:

- CLEAS
- Reclamo de Tercero - Taller
- Reclamo de Tercero - Abogado
- Recupero de Franquicia

## Etapa 7 - Cierre operativo

Entregables:

- reportes
- exportes avanzados
- optimizacion de permisos
- catalogos administrables
- ajustes de UX
- validacion funcional con usuarios reales

## Recomendaciones de ejecucion

- definir primero el modelo de datos comun
- separar reglas transversales de reglas por tramite
- trabajar con historias de usuario por solapa
- validar cada etapa con casos reales antes de sumar mas complejidad
